web github.com The second part of type-mapping primitives are a set of mappings that indicate what is allowed to map, based on element context. For example, an element might move between string , code , and id over the course of development. Understanding how primitives related to each-other is purely based on their "value domain" (the set of values that can be represented by the type). Primitive types do not hold the context required to evaluate a "concept domain" (what the contained value means ) - that context must be provided by the element that uses a type. The source-of-truth for the primitive mappings are defined in the input/types/ConceptMap-types-fallback.json ConceptMap, which applies across all versions.
