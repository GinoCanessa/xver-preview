Summary

Complex Extension: This cross-version extension represents the FHIR R5 element TestScript.setup.action.operation for use in FHIR R4.

  • type: Coding: Server interaction or operation type.
  • resource: uri: The type of the FHIR resource. See the resource list. Data type of uri is needed when non-HL7 artifacts are identified.
  • label: string: The label would be used for tracking/logging purposes by test engines.
  • description: string: The description would be used by test engines for tracking and reporting purposes.
  • accept: code: The mime-type to use for RESTful operation in the 'Accept' header.
  • contentType: code: The mime-type to use for RESTful operation in the 'Content-Type' header.
  • destination: integer: The server where the request message is destined for. Must be one of the server numbers listed in TestScript.destination section.
  • encodeRequestUrl: boolean: Whether or not to implicitly send the request url in encoded format. The default is true to match the standard RESTful client behavior. Set to false when communicating with a server that does not support encoded url paths.
  • method: code: The HTTP method the test engine MUST use for this operation regardless of any other operation details.
  • origin: integer: The server where the request message originates from. Must be one of the server numbers listed in TestScript.origin section.
  • params: string: Path plus parameters after [type]. Used to set parts of the request URL explicitly.
  • field: string: The HTTP header field e.g. "Accept".
  • value: string: The value of the header e.g. "application/fhir+xml".
  • requestId: id: The fixture id (maybe new) to map to the request.
  • responseId: id: The fixture id (maybe new) to map to the response.
  • sourceId: id: The id of the fixture used as the body of a PUT or POST request.
  • targetId: id: Id of fixture used for extracting the [id], [type], and [vid] for GET requests.
  • url: string: Complete request URL.

Maturity: 0

Summary

Complex Extension: This cross-version extension represents the FHIR R5 element TestScript.setup.action.operation for use in FHIR R4.

  • type: Coding: Server interaction or operation type.
  • resource: uri: The type of the FHIR resource. See the resource list. Data type of uri is needed when non-HL7 artifacts are identified.
  • label: string: The label would be used for tracking/logging purposes by test engines.
  • description: string: The description would be used by test engines for tracking and reporting purposes.
  • accept: code: The mime-type to use for RESTful operation in the 'Accept' header.
  • contentType: code: The mime-type to use for RESTful operation in the 'Content-Type' header.
  • destination: integer: The server where the request message is destined for. Must be one of the server numbers listed in TestScript.destination section.
  • encodeRequestUrl: boolean: Whether or not to implicitly send the request url in encoded format. The default is true to match the standard RESTful client behavior. Set to false when communicating with a server that does not support encoded url paths.
  • method: code: The HTTP method the test engine MUST use for this operation regardless of any other operation details.
  • origin: integer: The server where the request message originates from. Must be one of the server numbers listed in TestScript.origin section.
  • params: string: Path plus parameters after [type]. Used to set parts of the request URL explicitly.
  • field: string: The HTTP header field e.g. "Accept".
  • value: string: The value of the header e.g. "application/fhir+xml".
  • requestId: id: The fixture id (maybe new) to map to the request.
  • responseId: id: The fixture id (maybe new) to map to the response.
  • sourceId: id: The id of the fixture used as the body of a PUT or POST request.
  • targetId: id: Id of fixture used for extracting the [id], [type], and [vid] for GET requests.
  • url: string: Complete request URL.
