Extension: Cross-version Extension `R5.EvidenceReport.section` for use in FHIR R4

Official URL: section Version: 0.0.1-snapshot-3
Standards status: Trial-use Active as of 2026-02-21 Maturity Level: 0 Computable Name: ExtensionEvidenceReport_Section

This cross-version extension represents the FHIR R5 element EvidenceReport.section for use in FHIR R4.

Context of Use

    This extension does not specify which elements it should be used on

    This extension is allowed for use with FHIR versions R4 to R4 .

Usage info

Usages:

  • This Extension is not used by any profiles in this Implementation Guide

You can also check for usages in the FHIR IG Statistics

Summary

Complex Extension: This cross-version extension represents the FHIR R5 element EvidenceReport.section for use in FHIR R4.

  • title: string: The label for this particular section. This will be part of the rendered content for the document, and is often used to build a table of contents.
  • focus: CodeableConcept: A code identifying the kind of content contained within the section. This should be consistent with the section title.
  • focusReference: Reference: A definitional Resource identifying the kind of content contained within the section. This should be consistent with the section title.
  • author: Reference: Identifies who is responsible for the information in this section, not necessarily who typed it in.
  • _datatype: string: Slice to indicate the presence of a R5 Narrative in FHIR R4
  • status: code: The status of the narrative - whether it's entirely generated (from just the defined data or the extensions too), or whether a human authored it and it may contain additional data.
  • div: uri: The actual narrative content, a stripped down version of XHTML.
  • mode: code: How the entry list was prepared - whether it is a working list that is suitable for being maintained on an ongoing basis, or if it represents a snapshot of a list of items from another source, or whether it is a prepared list where items may be marked as added, modified or deleted.
  • orderedBy: CodeableConcept: Specifies the order applied to the items in the section entries.
  • entryClassifier: CodeableConcept: Specifies any type of classification of the evidence report.
  • entryReference: Reference: A reference to the actual resource from which the narrative in the section is derived.
  • entryQuantity: Quantity: Quantity as content.
  • emptyReason: CodeableConcept: If the section is empty, why the list is empty. An empty section typically has some text explaining the empty reason.
  • section: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: A nested sub-section within this section.

Summary

Complex Extension: This cross-version extension represents the FHIR R5 element EvidenceReport.section for use in FHIR R4.

  • title: string: The label for this particular section. This will be part of the rendered content for the document, and is often used to build a table of contents.
  • focus: CodeableConcept: A code identifying the kind of content contained within the section. This should be consistent with the section title.
  • focusReference: Reference: A definitional Resource identifying the kind of content contained within the section. This should be consistent with the section title.
  • author: Reference: Identifies who is responsible for the information in this section, not necessarily who typed it in.
  • _datatype: string: Slice to indicate the presence of a R5 Narrative in FHIR R4
  • status: code: The status of the narrative - whether it's entirely generated (from just the defined data or the extensions too), or whether a human authored it and it may contain additional data.
  • div: uri: The actual narrative content, a stripped down version of XHTML.
  • mode: code: How the entry list was prepared - whether it is a working list that is suitable for being maintained on an ongoing basis, or if it represents a snapshot of a list of items from another source, or whether it is a prepared list where items may be marked as added, modified or deleted.
  • orderedBy: CodeableConcept: Specifies the order applied to the items in the section entries.
  • entryClassifier: CodeableConcept: Specifies any type of classification of the evidence report.
  • entryReference: Reference: A reference to the actual resource from which the narrative in the section is derived.
  • entryQuantity: Quantity: Quantity as content.
  • emptyReason: CodeableConcept: If the section is empty, why the list is empty. An empty section typically has some text explaining the empty reason.
  • section: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: A nested sub-section within this section.
