Summary

Complex Extension: This cross-version extension represents the FHIR R5 element SubscriptionStatus for use in FHIR R4.

  • status: code: The status of the subscription, which marks the server state for managing the subscription.
  • type: code: The type of event being conveyed with this notification.
  • eventsSinceSubscriptionStart: string: The total number of actual events which have been generated since the Subscription was created (inclusive of this notification) - regardless of how many have been successfully communicated. This number is NOT incremented for handshake and heartbeat notifications.
  • eventNumber: string: Either the sequential number of this event in this subscription context or a relative event number for this notification.
  • timestamp: instant: The actual time this event occurred on the server.
  • focus: Reference: The focus of this event. While this will usually be a reference to the focus resource of the event, it MAY contain a reference to a non-FHIR object.
  • additionalContext: Reference: Additional context information for this event. Generally, this will contain references to additional resources included with the event (e.g., the Patient relevant to an Encounter), however it MAY refer to non-FHIR objects.
  • subscription: Reference: The reference to the Subscription which generated this notification.
  • topic: canonical: The reference to the SubscriptionTopic for the Subscription which generated this notification.
  • error: CodeableConcept: A record of errors that occurred when the server processed a notification.
