Summary

Complex Extension: This cross-version extension represents the FHIR R5 element ExampleScenario.process for use in FHIR R4.

  • title: string: A short descriptive label the process to be used in tables or diagrams.
  • description: markdown: An explanation of what the process represents and what it does.
  • preConditions: markdown: Description of the initial state of the actors, environment and data before the process starts.
  • postConditions: markdown: Description of the final state of the actors, environment and data after the process has been successfully completed.
  • number: string: The sequential number of the step, e.g. 1.2.5.
  • process: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: Indicates that the step is a complex sub-process with its own steps.
  • workflow: canonical: Indicates that the step is defined by a seaparate scenario instance.
  • type: Coding: The standardized type of action (FHIR or otherwise).
  • title: string: A short descriptive label the step to be used in tables or diagrams.
  • initiator: string: The system that invokes the action/transmits the data.
  • receiver: string: The system on which the action is invoked/receives the data.
  • description: markdown: An explanation of what the operation represents and what it does.
  • initiatorActive: boolean: If false, the initiator is deactivated right after the operation.
  • receiverActive: boolean: If false, the receiver is deactivated right after the operation.
  • request: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: A reference to the instance that is transmitted from requester to receiver as part of the invocation of the operation.
  • response: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: A reference to the instance that is transmitted from receiver to requester as part of the operation's synchronous response (if any).
  • title: string: The label to display for the alternative that gives a sense of the circumstance in which the alternative should be invoked.
  • description: markdown: A human-readable description of the alternative explaining when the alternative should occur rather than the base step.
  • step: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: Indicates the operation, sub-process or scenario that happens if the alternative option is selected.
  • pause: boolean: If true, indicates that, following this step, there is a pause in the flow and the subsequent step will occur at some later time (triggered by some event).
