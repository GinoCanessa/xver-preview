Summary

Complex Extension: This cross-version extension represents the FHIR R5 element OperationDefinition.parameter for use in FHIR R4.

  • name: code: The name of used to identify the parameter.
  • use: code: Whether this is an input or an output parameter.
  • scope: code: If present, indicates that the parameter applies when the operation is being invoked at the specified level.
  • min: integer: The minimum number of times this parameter SHALL appear in the request or response.
  • max: string: The maximum number of times this element is permitted to appear in the request or response.
  • documentation: markdown: Describes the meaning or use of this parameter.
  • type: code: The type for this parameter.
  • allowedType: code: Support for polymorphic types. If the parameter type is abstract, this element lists allowed sub-types for the parameter.
  • targetProfile: canonical: Used when the type is "Reference" or "canonical", and identifies a profile structure or implementation Guide that applies to the target of the reference this parameter refers to. If any profiles are specified, then the content must conform to at least one of them. The URL can be a local reference - to a contained StructureDefinition, or a reference to another StructureDefinition or Implementation Guide by a canonical URL. When an implementation guide is specified, the target resource SHALL conform to at least one profile defined in the implementation guide.
  • searchType: code: How the parameter is understood if/when it used as search parameter. This is only used if the parameter is a string.
  • strength: code: Indicates the degree of conformance expectations associated with this binding - that is, the degree to which the provided value set must be adhered to in the instances.
  • valueSet: canonical: Points to the value set or external definition (e.g. implicit value set) that identifies the set of codes to be used.
  • source: string: The name of the parameter or dot-separated path of parameter names pointing to the resource parameter that is expected to contain a reference to this resource.
  • sourceId: string: The id of the element in the referencing resource that is expected to resolve to this resource.
  • part: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: The parts of a nested Parameter.

Summary

Complex Extension: This cross-version extension represents the FHIR R5 element OperationDefinition.parameter for use in FHIR R4.

  • name: code: The name of used to identify the parameter.
  • use: code: Whether this is an input or an output parameter.
  • scope: code: If present, indicates that the parameter applies when the operation is being invoked at the specified level.
  • min: integer: The minimum number of times this parameter SHALL appear in the request or response.
  • max: string: The maximum number of times this element is permitted to appear in the request or response.
  • documentation: markdown: Describes the meaning or use of this parameter.
  • type: code: The type for this parameter.
  • allowedType: code: Support for polymorphic types. If the parameter type is abstract, this element lists allowed sub-types for the parameter.
  • targetProfile: canonical: Used when the type is "Reference" or "canonical", and identifies a profile structure or implementation Guide that applies to the target of the reference this parameter refers to. If any profiles are specified, then the content must conform to at least one of them. The URL can be a local reference - to a contained StructureDefinition, or a reference to another StructureDefinition or Implementation Guide by a canonical URL. When an implementation guide is specified, the target resource SHALL conform to at least one profile defined in the implementation guide.
  • searchType: code: How the parameter is understood if/when it used as search parameter. This is only used if the parameter is a string.
  • strength: code: Indicates the degree of conformance expectations associated with this binding - that is, the degree to which the provided value set must be adhered to in the instances.
  • valueSet: canonical: Points to the value set or external definition (e.g. implicit value set) that identifies the set of codes to be used.
  • source: string: The name of the parameter or dot-separated path of parameter names pointing to the resource parameter that is expected to contain a reference to this resource.
  • sourceId: string: The id of the element in the referencing resource that is expected to resolve to this resource.
  • part: base64Binary, boolean, canonical, code, date, dateTime, decimal, id, instant, integer, markdown, oid, positiveInt, string, time, unsignedInt, uri, url, uuid, Address, Age, Annotation, Attachment, CodeableConcept, Coding, ContactPoint, Count, Distance, Duration, HumanName, Identifier, Money, Period, Quantity, Range, Ratio, Reference, SampledData, Signature, Timing, ContactDetail, Contributor, DataRequirement, Expression, ParameterDefinition, RelatedArtifact, TriggerDefinition, UsageContext, Dosage, Meta: The parts of a nested Parameter.
