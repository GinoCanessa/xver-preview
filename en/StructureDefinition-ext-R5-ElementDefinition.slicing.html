Official URL: http://hl7.org/fhir/5.0/StructureDefinition/extension-ElementDefinition.slicing Version: 0.0.1-snapshot-3
Standards status: Trial-use Active as of 2026-02-17 Maturity Level: 0 Computable Name: ExtensionElementDefinition_Slicing

This cross-version extension represents the FHIR R5 element ElementDefinition.slicing for use in FHIR R4.

Context of Use

    This extension may be used on the following element(s)

    • Element ID: ElementDefinition.slicing

    This extension is allowed for use with FHIR versions R4 to R4 .

Usage info

Usages:

Summary

Complex Extension: This cross-version extension represents the FHIR R5 element ElementDefinition.slicing for use in FHIR R4.

  • type: code: How the element value is interpreted when discrimination is evaluated.
  • path: string: A FHIRPath expression, using the simple subset of FHIRPath, that is used to identify the element on which discrimination is based.
  • description: string: A human-readable text description of how the slicing works. If there is no discriminator, this is required to be present to provide whatever information is possible about how the slices can be differentiated.
  • ordered: boolean: If the matching elements have to occur in the same order as defined in the profile.
  • rules: code: Whether additional slices are allowed or not. When the slices are ordered, profile authors can also say that additional slices are only allowed at the end.

Maturity: 0
